How To Set-Up A Web Server With Docker

Step 1:

Install Docker Engine (https://docs.docker.com/engine/install) (https://docs.docker.com/get-docker/) and make sure it is running and working properly. A good way to do this would be to run the command "docker run hello-world"!

Running hello-world to test Docker.

Step 2:

Pull an Ubuntu image using the command "docker pull ubuntu" (shown below). Then make sure this image is installed and working by doing the command "docker images", which shows all of the images installed (shown below).

Pulling Ubuntu image and checking to see if it is there.

Step 3:

Part 1:

Run your Ubuntu image with the bash shell by running the command "docker run -it ubuntu:latest /bin/bash". After, update Ubuntu by running "apt-get update". Then install apache2 by running "apt-get install apache2". Finally, install vim by running "apt-get install vim". Then type "exit" to exit.

Updating Ubuntu to the latest version.

Installing apache2, which is what is going to run our web server.

Installing vim to create/edit files.

Part 2:

After those tasks are complete. Run "docker ps -a", and find the container that you were just in. Once you do this you need to commit by running "docker commit (container ID) (new name)". Finally, run "docker images" to make sure your new image, with its new name, is there.

Changing the Ubuntu image name to something more recognizable.

Step 4:

For this step, you need to bind a directory in the container to a directory on your local machine. This can be done by running the command "docker run -it -p 127.0.0.1:80:80 -v (Local Directory Path):(Container Directory Path) ubuntu-my_apache2 /bin/bash" (shown below).

Binding my local directory to the /var/www/html directory in the container

Step 5:

Log into your Ubuntu container again by running the command "docker run -it ubuntu:latest /bin/bash". Once inside of the container, change directory (cd) to the directory that you just binded in the previous step. Once inside of the directory, create an index.html file that will be used to test the web server. You can write what ever you please on this file, as it is only used to test that the web server is working.

This is the code for my test file (index.html).

This shows the index.html in my local directory.

Step 6:

After doing the previous step to create a test file, you now should restart apache2 from inside of the Ubuntu container by running the command "/etc/init.d/apache2 restart". After you have restarted apache2, visit 127.0.0.1 to see your live website, which is using your test file from earlier as the homepage.
